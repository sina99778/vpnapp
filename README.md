# SecureVPN Middleware 🛡️

سیستم مدیریت و اتوماسیون هوشمند فروش VPN بر پایه معماری **Zero-Trust**. این پروژه به عنوان یک لایه میانی بین کلاینت‌ها (کاربران) و هسته‌ی پنل‌های مدیریت VPN (مانند **Marzban** و **Remnawave**) قرار می‌گیرد و با جداسازی اطلاعات حساس و مدیریت یکپارچه بار ترافیک، پایداری و امنیت شبکه را تضمین می‌کند.

## ✨ قابلیت‌ها (Features)

*   **معماری Zero-Trust**: هسته پنل‌های VPN هیچ‌گاه دیتای مربوط به کاربران واقعی یا پرداخت‌ها را نمی‌بینند. تمام اکانت‌های پنل به صورت `ephemeral` (یکبارمصرف) با UUID های تصادفی ایجاد می‌شوند.
*   **پشتیبانی از چند پنل (Multi-Panel)**: اتصال و پشتیبانی یکپارچه از سیستم‌های مبتنی بر **Marzban / Rebecca** و پنل مدرن **Remnawave** تنها با تغییر یک متغیر در فایل `.env`.
*   **مدیریت هوشمند قطعی‌ها (Circuit Breaker)**: اگر سروری در لیست نودها دچار قطعی شود یا توانایی ساخت کاربر جدید را نداشته باشد، سیستم به صورت خودکار متوجه آن شده و آن سرور را از مسیر نوبت‌دهی (Load Balancing) خارج می‌کند تا کاربران متصل دچار "سیاه‌چاله" نشوند.
*   **حفاظت در برابر Replay Attack**: یکپارچگی کامل با سیستم App Attest شرکت اپل و Play Integrity گوگل.
*   **سیستم تبلیغات و پاداش (Ads & Rewards)**: امکان اضافه کردن زمان رایگان با مشاهده تبلیغات (برای اکانت‌های Free).

## 📥 نصب و راه‌اندازی با یک خط کد (One-Line Installer)

برای نصب سریع و بدون دردسر روی سرور خود، کافیست دستور زیر را وارد کنید. این اسکریپت تمامی نیازمندی‌ها (مانند Docker) را بررسی کرده، ریپازیتوری را دانلود می‌کند، پسوردهای قدرتمند و رندوم برای دیتابیس می‌سازد و سرویس‌ها را اجرا می‌کند:

```bash
curl -fsSL https://raw.githubusercontent.com/sina99778/vpnapp/main/install.sh | bash
```

**در هنگام نصب از شما پرسیده می‌شود:**
1. آدرس عمومی وب‌سرویس شما چيست؟
2. مایلید از پنل `rebecca` استفاده کنید یا `remnawave`؟
3. اطلاعات دسترسی و توکن پنل (بسته به انتخاب مرحله قبل).

### ⚙️ نصب دستی (Manual Installation)

اگر مایل هستید نصب را به صورت دستی انجام دهید:

1. پروژه را Clone کنید:
   ```bash
   git clone https://github.com/sina99778/vpnapp.git
   cd vpnapp
   ```
2. اسکریپت نصب را با پارامتر مربوطه اجرا کنید تا فقط تنظیمات انجام شود و داکر اجرا نگردد:
   ```bash
   ./install.sh --no-start
   ```
3. فایل تنظیمات را باز کرده و در صورت نیاز ویرایش کنید (`backend/.env` و `.env`).
4. کانتینرها را بیلد و اجرا نمایید:
   ```bash
   docker compose up -d --build
   ```

## 🛠 ابزارها و پنل‌های پشتیبانی شده

### اتصال به Remnawave
برای استفاده از Remnawave در سیستم کافیست در `.env` بک‌اند موارد زیر تنظیم شده باشد:
```dotenv
PANEL_PROVIDER=remnawave
REMNAWAVE_BASE_URL=https://your-panel.com
REMNAWAVE_TOKEN=YOUR_API_TOKEN
REMNAWAVE_SQUAD_UUIDS=squad-id-1,squad-id-2
```

### اتصال به Marzban (Rebecca)
برای استفاده از فورک‌های مرزبان:
```dotenv
PANEL_PROVIDER=rebecca
PANEL_BASE_URL=https://panel.internal:8000
PANEL_ADMIN_USER=admin
PANEL_ADMIN_PASS=secret
```

## 📚 مدیریت سرویس‌ها

- **بررسی لاگ‌های API**:
  ```bash
  docker compose logs -f api
  ```
- **بررسی لاگ‌های ورکر همگام‌سازی نودها**:
  ```bash
  docker compose logs -f node-sync
  ```
- **ریستارت کردن یک سرویس خاص** (مثلا API):
  ```bash
  docker compose restart api
  ```
- **پاک کردن کامل دیتابیس و کانتینرها**:
  ```bash
  docker compose down -v
  ```

## 👨‍💻 معماری دیتابیس
این پروژه از PostgreSQL استفاده می‌کند. برای جلوگیری از مشکلات تداخل دیتابیس در هنگام سوئیچ بین پنل‌ها (نظیر تفاوت شناسه گره‌ها در Marzban و Remnawave)، شناسه نود (`panel_node_id`) از نوع `text` در نظر گرفته شده است تا با هر دو پنل مطابقت داشته باشد.

توسعه یافته با ❤️ و معماری NestJS.
